How to get child Maven Module to use Parent Project Properties
--------------------------------------------------------------
  
2. How to set the version number
   mvn versions:set -DnewVersion=2.5.0-SNAPSHOT
   
   *or*
   
   mvn versions:set -DnewVersion=2.5.1-SNAPSHOT -DgenerateBackupPoms=false
  
  
  
3. PROBLEM:  mvn appears to be ignoring your pom.xml settings
   SOLUTION  Make sure you are in the *CORRECT* directory that you think you are
   
    
4. PROBLEM:  your child project cannot find references to the parent project
   SOLUTION: Make sure your child pom.xml has a reference to the parent
                 <parent>
                       <artifactId>...</artifactId>
                       <groupId>...</groupId>
                       <version>...</version.
                </parent>
                
               
               So, if your parent pom.xml has properties
                   <properties>
                      <slf4j-api-version>1.7.7</slf4j-api-version>
                    </properties>
                    
              Then, your child pom.xml can use them in the regular
                 <dependencies>
                    <dependency>
                       <groupId>org.slf4j</groupId>
                       <artifactId>slf4j-api</artifactId>
                       <version>${slf4j-api-version}</version>
                    </depenendy>
                  </dependencies>
