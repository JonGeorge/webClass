How to Password Protect the Grub Menu
-------------------------------------

If your Grub menu is unprotected, then a user (with physical access to your centos 7 computer) can set your root password.

References
----------
https://www.tecmint.com/things-to-do-after-minimal-rhel-centos-7-installation/6/#C30

Procedure
---------
 1. Backup your grub files
    unix> sudo -s
    unix> cp /boot/grub2/grub.cfg /boot/grub2/grub.cfg.old
    unix> cp /etc/grub.d/10_linux /etc/grub.d/10_linux.old

 2. Add this to the end of your 10_linux file 
    unix> vi /etc/grub.d/10_linux
    
    cat <<EOF
    set superusers=”johnsmith”
    Password johnsmith secret
    EOF
    
    NOTE:  replace “tecmint” as username and “avi@123” as password with your username and password.
    
    
 3. Generate a new grub.cfg file
    unix>  grub2-mkconfig --output=/boot/grub2/grub.cfg

 4. Reboot
    unix> reboot
    
 5. Attempt to edit the grub menu
    -- You will be prompted for your username/password
